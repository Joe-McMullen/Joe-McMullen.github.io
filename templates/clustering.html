Identify Similar Players using Clustering for Recruitment

Clustering and valuing players and Using Data from Football Manager and Transfermarkt

Stack: Python, Tableau

This project aims to use data from Football Manager 2023 to group players with a similar profile. The project also aims to use data from Transfermarkt to create a player valuation model based on their attributes. These will be combined into an application which can take a player as input and produchort-list of similar playere a ss with their profile, including a valuation. This may be useful for recruitment departments who want to maximise the resources of their scouting departments by using data to identify relevant transfer targets for further analysis.

Data

Football Manager breaks player attributes down into 3 categories (technical, mental, physical) and these are further seperated into 36 attributes as seen below. These attributes are based on real world performance and provide a broad overview of the qualities of a player. Player attributes as well as meta data was scraped for 1045 players from FMInside using BeautifulSoup and selenium within python.

Transfermarkt is a website which provides player values based on community opinion. This provides valuations which take into account many factors beyond those captured in football metrics including marketing value and injury susceptibility. The dataset used in this project was downloaded from kaggle and is regularly updated.

Modelling

In this project, player similarity was determined using K-means clustering on the 36 attributes in the Football Manager dataset shown above. Within Football Manager, goalkeepers have a different set of attributes to outfield players, as a result, they were excluded from the dataset. This optimum clustering was observed with 18 different player profiles considering the inertia and silhouette coefficient for differing numbers of clusters. The below visualisation of this clustering was produced using the t-SNE algorithm for dimensionality reduction.

The datasets containing player attribrutes and values were combined to find patterns between the attributes and valuations. A linear regression model was fit to estimate player value. The key features with the most influence on value were found to be be age, stamina, pace, and acceleration. This suggests a premium for young, and athletically promising players which might be expected given they will be considered to have high potential value in the future. These models were combined to create a demo in Tableau which for a selected player (Kevin De Bruyne in this case), the 10 most similar players were identified and information and their comparisons are displayed to the user. Recruiting departments could use this application to identify player replacements or alternative options once a transfer target is determined. This could aid in reducing time spent scouting unsuitable players, helping to increase efficiency of the recruiting department.

Improvements and Future Work

Whilst Football Manager attributes are based on real world data, improvements to this project could be made by directly building up metrics from real world event and tracking data. This will allow for more bespoke metric selection which may be useful in targetting specific profiles. Improvements could also be made in the data processing, with normalisation being used to ensure clustering focussed on players with a similar overall profile are grouped rather than a similar quality level. Post-processing such as weightings based on a players proximity to cluster centres may also give more explainable output regarding the profile of a player.

The use of different clustering algorithms or classifiers for value estimation should also be investigated as well as the inclusion of more parameters. For example, player information such as nationality or league may be relevant to value modelling.
