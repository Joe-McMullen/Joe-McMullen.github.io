Football Passing Networks Weighted with Expected Threat

Combining advanced metrics and network science approaches to gain insight into team performance and player roles

Stack: Python, Networkx, MongoDB

This project aims to develop an Expected Threat(xT) model and use the xT of passes as weights in a team's passing network to gain insight into the influence of players to attacking phases of team's play. xT is a statistical model which estimates the likelihood of scoring within the current posession based on location on the pitch. Using this, events which move the ball such as passes or dribbles can be given a value based on the increase in likelihood of scoring between the location where the event started and finished.

Passing networks (pictured below) can be used to gain insight into the connections and structure of a team when in possession. Traditionally a football passing network considers the players of a team as nodes located in the average position of their passes made. The connections(edges) between two players represent the volume of passes between them. This project instead considers the xT created by passes between players as the edges to determine which combinations are most important to advancing the ball into more dangerous positions. Using network measures, the pass networks produced can be analysed to assess team performance and player role within the team system.

Approach

The expected threat algorithm used in this project follows the implementation of Karun Singh of Karun Singh who's work is based on Sarah Rudd's research first describing football in terms of a markov model. The pitch is broken down into (16 x 12) tiles and in each tile, there is the possibility to shoot, pass, or dribble (pass and dribble will be grouped as "move" actions herein).

The reward for shooting in a given tile is the ratio of goals scored to the number of shots taken in that tile (as in a simple expected goals(xG) model).

In Sarah Rudd's implementation, the value of moving to a tile is achieved at the convergence of the transition matrix of the markov chain model. The model implemented here does not feature an absorption state in the form of possession being lost. Only successful actions are considered. As a result, if left to run until convergence, each tile move reward will rise to that of the average shot value giving us a poor model. For this reason, the model was iterated 5 times. This gives the probability that from a given location, a goal will be scored within 5 successful events.
